# Contributing

Thanks for improving First Frame Agent.

## Before You Open a PR

1. Read [README.md](README.md) and the project source.
2. Keep the change narrow. This repo should continue to do one job well.
3. Run the documented development checks.
4. Update README examples when behavior changes.

## Pull Request Shape

- Explain the problem and the fix.
- Include the command output for the checks you ran.
- Avoid unrelated formatting churn.
- Do not add secrets, generated dependency folders, or private data.

## Issues

Good issues include expected behavior, actual behavior, reproduction steps, and
environment details. Broad product requests may be closed if they push the repo
outside its narrow scope.
